Git 2.0

By Chao Pang 2021-07-28

為響應黑人平權運動，GitHub宣布從2020/10月起改變新Git儲存庫的預設命名，以main來取代原本的master。

1. 下載現有的遠端儲存庫Github的repo git clone https://github.com/chaoban/hahowgit.git

2. 或者，在本地端新建一個儲存庫，並和遠端儲存庫Github的repo做連結

  • 新建立一個檔案並commit到本地儲存庫(repo)內。

    echo "# hahowgit" >> README.md
git init
git add README.md
git commit -m "first commit"

  • 將本地儲存庫和遠端儲存庫Github的repo做連結 (補充: 下面指令執行時，可搭配git remote show origin看每一階段的訊息)

​ 1. 新建一個名稱為main的branch

​ git branch -M main

​ 2. 新增一個叫做origin的遠端儲存庫，它的url為https://github.com/chaoban/hahowgit.git

​ git remote add origin https://github.com/chaoban/hahowgit.git

​ 3. 把現在所在的main這個branch的資料，推到origin這個儲存庫內 (補充: HEAD是工作目錄現在所在的commit)

​ git push -u origin main

3. 又或者，將現有的本地儲存庫和遠端儲存Github的repo做連結

git remote add origin https://github.com/chaoban/hahowgit.git
git branch -M main
git push -u origin main

4. 將本地儲存庫的資料(已經commit)上傳到遠端儲存庫Github

git push

push an existing repository from the command line
git remote add origin https://github.com/chaoban/test_github.git
git branch -M main
git push -u origin main

git remote add origin https://github.com/chaoban/test_github.git 用remote意味著對遠端repoistory操作，新增origin到repo內。

git branch -M main 新增一個branch，叫做main。

git push -u origin main 把main這個branch的資料推到origin內。

自從 git 升級為 2.x 版本之後，push 模式就調整為兩種模式 Matching、Simple 如果你沒有設定，在push 就現出以下警告訊息

warning: push.default is unset; its implicit value is changing in Git 2.0 from ‘matching’ > to ‘simple’. To squelch this message and maintain the current behavior after the default > changes, use:

​ git config –global push.default matching

To squelch this message and adopt the new behavior now, use:

​ git config –global push.default simple

Matching

​ Push 設定為 upstream 的 branch

Simple

​ Push 當下所在的 branch，其餘 branch 不會push

在 git 2.0 之前，預設使用的是 Matching 在 git 2.0 後，我是設定

​ git config --global push.default simple

如果需要 push all branch 可以使用--all 參數

$ git push --all
