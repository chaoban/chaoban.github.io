ROS概念和指令

By Chao Pang 2021-07-27

這是我閱讀一書 ROS Robot Programming(A Handbook Written by Turtlebot3 Developers) 的筆記，主要在第四章和第五章。

概念

Master

負責節點到節點的連接和消息通信。類似一個Name Server。roscore是它的運行命令。

Node

最小運行處理單元，可以視為一個執行程式。

Package

構成ROS的基本單元。ROS應用程式是以Package為單位開發的。

Metapackage

具有共同目的的Package的集合。例如導航Metapackage，包含了AMCL，DWA，EKF…等10多種Package。

Message

  1. Topic 單向，異步

  2. Service 雙向，同步

  3. Action 雙向，異步

    1. Goal 目標
    2. Result 結果
    3. Feedback 回饋
    Topic

    為單向的消息發送與接收。Publisher和Subscriber，用Topic發送和接收訊息。

    Service

    為雙向的消息請求與回應。

    Action

    為雙向消息Goal、Result、與Feedback。

指令

  1. Shell命令
    1. roscd

    切換到ros功能包的目錄。

    roscd [功能包名稱]

    $ roscd turtlesim

  2. 運行命令

    1. roscore

      執行主節點。主節點管理節點之間的訊息通訊中的連結訊息。包括master（ROS名稱服務）+ rosout（日誌）+ parameter server（參數管理）。ROS_MASTER_URI作為主節點的URI，在~/.bashrc中設置。

    2. rosrun

      運行ROS節點。 rosrun [功能包名稱] [節點名稱]

      $ rosrun turtlesim turtlesim_node

    3. roslaunch

      運行多個ROS節點，及設定運行選項。

      roslaunch [功能包名稱] [launch檔案名稱]

      $ roslaunch openni_launch openni.launch

    4. rosclean

      檢查或刪除ROS日誌。

      rosclean [選項]

      $ rosclean check

      $ rosclean purge

      Log files位於/home/rt/.ros/log

  3. 訊息命令
    1. rosnode
    2. rostopic
    3. rosservice
    4. rosparam
    5. rosmsg
    6. rossrv
    7. rosbag
  4. catkin命令
    1. catkin_create_pkg
    2. catkin_make
    3. catkin_init_workspace
  5. 功能包命令
    1. rospack
    2. rosinstall
    3. rosdep
